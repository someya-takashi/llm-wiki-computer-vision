# WIT（WebImageText, 400M）

## 概要

**WIT** = **WebImageText**。OpenAI の Radford ら（2021）が CLIP（[[entities/clip]]）の事前学習のために構築した、**4 億組の（画像, テキスト）ペア**からなる大規模ウェブデータセット。CV における初の本格的な web-scale 画像-テキストデータセットで、後の LAION や PE のデータセットの先駆け。

- 出典: [[sources/clip]] §2.2
- 規模: **4 億**（image, text）ペア
- 総単語数: GPT-2 の WebText データセットと同程度
- 公開状況: **非公開**（OpenAI 内部データセット）

> **重要**: 「**WIT**」という名前は Google の WIT（Wikipedia-based Image Text）データセットとは別物なので注意。CLIP の WIT は OpenAI 製で非公開。

---

## なぜ重要か

WIT は CV における「データのスケール」を一気に押し上げた：

| データセット | 規模 | 種類 |
|---|---|---|
| ImageNet-1k | 128 万 | 教師ありラベル |
| ImageNet-22k | 1400 万 | 教師ありラベル |
| MS-COCO | 約 10 万 | クラウドソースキャプション |
| Visual Genome | 約 10 万 | クラウドソース注釈 |
| YFCC100M | 1 億 | ノイズメタデータ |
| YFCC100M (英語フィルタ後) | 1500 万 | ノイズキャプション |
| JFT-300M (Google) | 3 億 | ノイズラベル |
| Instagram WSL | 35 億 | ハッシュタグ |
| **WIT** | **4 億** | **自然言語キャプション** |

これにより、**「ウェブから自然言語の教師信号を集める」**ことの実用性を実証し、後続を生んだ：

- **LAION-400M** (2021): WIT のオープンソース対応物
- **LAION-2B / LAION-5B** (2022): OpenCLIP の訓練データ
- **WebLI** (Google, 内部): SigLIP の訓練データ
- **MetaCLIP** (Meta, 2023): WIT のキュレーション再現
- **DataComp** (2023): ウェブ画像-テキストのベンチマーク化
- **PE-Pretraining** (Meta, 2024): 860 億ペア

## 構築方法（§2.2）

CLIP 論文では構築方法を簡潔にしか述べていないが、要点：

### Step 1: クエリ集合の作成

- **50 万の検索クエリ**を作成
- 単語頻度や Wikipedia のリンクなどから派生（詳細は論文不明）
- 視覚的概念の幅広いカバレッジを目指す

### Step 2: 画像-テキストペアの収集

- 各クエリに対して、テキストがそのクエリを含むようなウェブ上の（画像, テキスト）ペアを検索
- **各クエリで最大 20,000 ペア**まで（クラスバランスのため）

### Step 3: 結果

- 50 万クエリ × 最大 2 万 = 最大 100 億の理論値
- 実際には **4 億ペア**に落ち着いた

> **補足: 「クラスバランス」の意味** — Web からそのまま集めると「犬・猫」のような popular 概念に偏る。各クエリで上限を設けることで、「希少な視覚的概念も一定量含む」ようにバランスを取った。これが後の LAION との品質差の一因。

## WIT の特徴（既存データセットとの違い）

### vs YFCC100M

- YFCC100M はメタデータが疎で品質低い（カメラ設定や `20160716_113957.JPG` のような自動ファイル名）
- 英語フィルタ後 1500 万に縮小、ImageNet サイズ
- WIT はクエリで品質と多様性を担保

### vs Instagram WSL（[Mahajan 2018]）

- WSL は 35 億 Instagram 画像をハッシュタグで学習
- ハッシュタグ集合は事前定義されたカテゴリ（〜18000）
- WIT は**自由形式の自然言語**

### vs JFT-300M

- JFT は 3 億 Google 内部画像、ノイズラベル（18291 クラス）
- WIT は固定クラス概念がない
- WIT の方がより**柔軟**

### vs MS-COCO / Visual Genome

- COCO は高品質クラウドキャプション、5 文 × 12 万画像
- WIT は低品質だが**4000 倍大規模**

## アブレーション: WIT vs YFCC100M（Appendix D）

CLIP 論文は WIT と同サイズの YFCC100M サブセットで比較実験を行った：

| | Linear Probe 平均 | Zero-Shot 平均 |
|---|---|---|
| YFCC100M | 65.5 | 29.6 |
| **WIT** | **66.6** | 30.0 |
| Δ | -1.1 | -0.4 |

平均では**ほぼ同等**だが、データセット固有の性能では大きな違い：

| データセット | YFCC > WIT | WIT > YFCC |
|---|---|---|
| Birdsnap | +12.1 | |
| Flowers102 | +4.6 | |
| Country211 | +5.8 | |
| Cars | | -18.9 |
| UCF101 | | -5.7 |
| GTSRB | | -5.7 |

**解釈**: 各データセットの性能は「事前学習データに関連する画像がどれだけ含まれているか」を反映する。YFCC100M は写真家が撮る花や鳥が多く、WIT は車やペットが多い、というデータ分布の違い。

→ **重要な含意**: CLIP のアプローチは「合理的にフィルタリングされた任意の画像-テキストペアコレクション」で動く。**WIT 自体に魔法はない**、サイズが鍵。

## なぜ非公開か

OpenAI は WIT を一度も公開していない。推測される理由：

1. **法的問題**: ウェブから収集した画像の著作権
2. **プライバシー**: 個人の画像が含まれる可能性
3. **商業的優位**: OpenAI の競争力の一部
4. **規模・配布の困難**: 4 億画像のホストは現実的でない

CLIP コードと事前学習済みモデルは公開されているが、データは公開せず、というスタンス。

→ 結果として **LAION** などのオープンソース代替が生まれ、OpenCLIP の隆盛につながった。

---

## 倫理的・社会的懸念

CLIP 論文 §7 で著者ら自身が言及：

- **キュレーションなし → 社会的バイアスを継承**: 西洋偏重、性別・人種偏重
- **個人の画像**: 同意なく訓練に使われる
- **危険なコンテンツ**: フィルタリングしているが完璧でない
- **CSAM 等の懸念**: 後の LAION-5B で実際に発見された問題

これらの問題は後続データセット（LAION 等）でより顕在化し、データ収集の倫理に関する議論を加速させた。
